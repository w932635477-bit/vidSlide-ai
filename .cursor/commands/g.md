# g - VidSlide AI 约束检查命令

**功能**: 检查代码是否符合.cursor-constraints.md约束文档
**使用方法**: 在聊天中输入 /g 即可调用
**输出**: 仅列出违反约束的项目和修正建议，不生成代码

## 检查规则清单

### Rule 1（测试覆盖）
**规则内容**: 所有功能必须编写对应的单元测试文件（xxx.test.js）
**适用范围**: All source files
**检查要点**:
- 新功能是否有对应的测试文件
- 测试覆盖率是否≥80%
- 测试是否涵盖核心业务逻辑

### Rule 2（错误处理）
**规则内容**: 所有异步操作必须有try/catch，所有函数必须处理错误
**适用范围**: *.js, *.vue files
**检查要点**:
- 异步操作是否有错误处理
- Promise链是否有catch
- 用户友好的错误提示

### Rule 3（技术栈合规）
**规则内容**: 必须使用技术文档指定的技术栈
**适用范围**: All files
**检查要点**:
- 是否使用了禁止的技术栈（React、Vuex等）
- 是否添加了未授权的依赖
- 是否符合当前阶段的开发约束

### Rule 4（功能范围控制）
**规则内容**: 禁止实现技术文档未提及的功能
**适用范围**: All feature implementations
**检查要点**:
- 是否符合当前开发阶段要求
- 是否有多余的功能实现
- 是否严格按照README.md需求

### Rule 5（代码质量）
**规则内容**: ESLint 0错误，符合代码风格规范
**适用范围**: *.js, *.vue files
**检查要点**:
- ESLint检查是否通过
- 代码注释率≥20%
- 命名规范（大驼峰、小驼峰等）

## 检查流程
1. 分析当前代码文件
2. 逐条验证规则合规性
3. 列出所有违反的项目
4. 提供具体的修正建议
5. 确认无违反后才能继续开发
